----------------------------Oracle数据库-----------------------------------------------------------
--查询语句逻辑顺序
1.在PGA保留该SESSION的用户连接信息和权限信息
2.生成一条HASH值，查找共享池中是否存在相同的HASH值
3.检查SQL语句语法是否正确，是否有查询权限，HASH值保存在共享池中
4.解析SQL语句，根据代价生成执行计划，并与HASH值关联（索引扫描或值班表扫描）
5.在数据缓冲区中查找数据，没有则查找磁盘中数据库文件将数据带回到数据缓冲区
6.显示查询结果
--更新语句逻辑顺序
1.在PGA保留该SESSION的用户连接信息和权限信息
2.生成一条HASH值，查找共享池中是否存在相同的HASH值
3.检查SQL语句语法是否正确，是否有查询权限，HASH值保存在共享池中
4.解析SQL语句，根据代价生成执行计划，并与HASH值关联（索引扫描或值班表扫描）
5.在数据缓冲区中查找数据，没有则查找磁盘中数据库文件将数据带回到数据缓冲区
6.在回滚表空间中分配事务槽，在缓冲区中生成更新前的数据镜像，由CKPT决定将数据镜像写入磁盘的数据文件，并生成日志
7.commit修改数据
7.将更新操作写入日志文件
8.CKPT触发启动DBWR将数据从缓冲区写入磁盘
9.rollback回滚数据，将数据从回滚段中读取出数据镜像还原到数据缓冲区，并写入日志，完成回滚



从磁盘读取数据块到内存的操作叫物理读
从Buffer Cache中提取数据的过程，便是逻辑读
DBWR:将数据从数据缓冲区保存到磁盘数据文件，必须LGWR先记录日志，才将数据写入磁盘
LGWR：将日志从日志缓冲区保存到磁盘REDO日志文件,LGWR故障停止会导致DBWR停止，因为任何操作皆需要日志
ARCH：在日志文件空间写满后将日志归档到其它存储介质
CKPT：CKPT参数FAST_START_MTTR_TARGET决定什么时间启动DBWR将数据存到磁盘上
PMON：进程监视器，控制数据库后台进程异常时的动作
SMON：系统监视器，监控系统级操作，清理临时表空间，清理回滚段表空间等
LCKn:仅用于RAC数据库，用于实例间的封锁
RECO：用于分布式数据库的恢复，发生于多个数据库之间的事务，事务失败就全部回滚
COMMIT提交是触发LGWR运行一次，将操作步骤从内存中写入磁盘的日志文件，并不意味着数据已经插入数据库，真正插入数据库的动作由进程CKPT决定，具体受参数FAST_START_MTTR_TARGET控制

每隔3秒LGWR运行一次
任何COMMIT触发LGWR运行一次
DBWR要把数据缓存写到磁盘，触发LGWR运行一次
日志缓冲区满三分之一或记录满1MB，触发LGWR运行一次
联机日志文件切换也将触发LGWR一次

数据库->数据表（系统表空间、回滚段表空间、临时表空间、用户表空间）->区->块
块：数据块头、表目录区、行目录区、可用空间区、行数据区,块是数据库最小的空间单位
区：区是数据库最小的分配空间单位，即系统分配分配空间按区为单位，不会一个块一个块分配空间
表：普通表、全局临时表、外部表、分区表、索引组织表、簇表等
分区表：范围分区、列表分区、HASH分区、组合分区

当连接查询没有where条件时，左连接查询时，前面的表是驱动表，后面的表是被驱动表，右连接查询时相反，内连接查询时，哪张表的数据较少，哪张表就是驱动表
当连接查询有where条件时，带where条件的表是驱动表，否则是被驱动表
如果where后面两张表的条件都有，那么驱动表的选取原则是和没有where条件的选取原则是一样的

如果a字段查询的记录比较多，b字段的查询的记录也比较多，而a和b同时查询的记录比较少，就适合建联合索引
过多的字段建联合索引不可取，因为这样索引也必然过大，影响性能，一般不宜超过3个字段组合
联合索引在等值查询情况下，组合索引的列无论哪个字段写在where子句前面，性能都一样
组合索引的两列，当一列是范围查询，一列是等值查询的情况下，等值查询列写在前，范围查询列在后，这样性能高


嵌套循 环连接驱动表的顺序，小的结果集先访问，大的结果集后访问，才能保证被驱动表的访问次数降到最低，从而提高性能
嵌套循环连接和哈希连接有驱动顺序，驱动表的顺序不同将影响表连接的性能，而排序合并连接没有驱动的概念，无论哪张表在前面都无妨
排序连接不支持<>的连接条件，也不支持LIKE的连接条件，但是比起HSAH连接，支持面要广一些，支持>之类的连接条件
嵌套循环连接所有的SQL连接条件写法，没有任何限制
嵌套循环连接要在驱动表的限制条件加索引，在被驱动表的连接条件加索引
外键未建索引更新相关表时将引发死锁及两表关联查询时性能低下
命名规则：表以t_为前缀，视图以v_为前缀，同义词以s_为前缀，簇表以c_为前缀，序列以seq_为前缀或后缀，存储过程以p_为前缀，函数以f_为前缀，类以typ_为前缀，主键以pk_为前缀，外键以fk_为前缀，唯一索引以ux_为前缀，普通索引以idx_为前缀，位图索引以bx_为前缀，函数索引以fx_为前缀
数据库应用结构：1.单磁盘独立主机结构：一个数据库服务器，一个数据库结构（数据库文件），即一台计算机，一个硬盘
                2.多磁盘独立主机结构：一台计算机多个硬盘，利用磁盘镜像技术，每个磁盘都有其它盘的完整备份，一个磁盘损坏可用其它磁盘代替或恢复数据
				3.多数据库独立主机结构：一台计算机，一个或多个硬盘，一台计算机安装多个版本Oracle数据库，数据库之间独立不共享
				4.C/S结构：客户机——交换机——数据库服务器
				5.B/S结构：客户机——Web服务器——Oracle数据库
				6.分布式结构：数据存储在多个服务器上，由通信管理、全局数据库管理系统、局部数据库管理系统进行管理
数据控制语言有GRANT（授权）、REVOKE（回收）、COMMIT（提交）、ROLLBACK（回滚）等
视图和基表构成关系数据库的外模式结构，数据库的存储文件和索引文件构成关系数据库的内模式
聚簇索引即索引项的顺序与表中记录的物理顺序相一致的索引组织
由于聚簇索引是将索引和表记录放在一起存储，所有在一个基表上最多只能建立一个聚簇索引。

union取合集并过滤重复数据，union all 取合集不过滤重复数据，intersect取交集（两个表中都有数据），minus取差集（取A-B的记录）
truncate tableName resuer storage  删除表空间后仍然保存记录占用的空间
truncate tableName drop storage    删除表空间后立即回收记录占用的空间，默认为此方式
truncate删除比delete快，因为truncate不会产生回滚信息，但也因此truncate操作不能被撤销

会话状态：
ACTIVE   ：处于此状态的会话，表示正在执行，处于活动状态。
INACTIVE ：处于此状态的会话表示不是正在执行的
KILLED   ：处于此状态的会话，表示出现了错误或进程被杀掉，正在回滚，当然，这个状态的会话也占用系统资源的。还有一点就是，    KILLED的状态一般会持续较长时间，如果你想快速杀掉回话，可以参考我以前的一篇文章ORACLE快速彻底Kill掉的会话

SNIPED   ： Session inactive, waiting on the client。 标记为SNIPED的进程被释放有两种条件：
         1、相关的terminal再一次试图登录及执行sql

SELECT Upper(F.TABLESPACE_NAME)         "表空间名",
       D.TOT_GROOTTE_MB                 "表空间大小(M)",
       D.TOT_GROOTTE_MB - F.TOTAL_BYTES "已使用空间(M)",
       To_char(Round(( D.TOT_GROOTTE_MB - F.TOTAL_BYTES ) / D.TOT_GROOTTE_MB * 100, 2), '990.99')
       || '%'                           "使用比",
       F.TOTAL_BYTES                    "空闲空间(M)",
       F.MAX_BYTES                      "最大块(M)"
FROM   (SELECT TABLESPACE_NAME,
               Round(Sum(BYTES) / ( 1024 * 1024 ), 2) TOTAL_BYTES,
               Round(Max(BYTES) / ( 1024 * 1024 ), 2) MAX_BYTES
        FROM   SYS.DBA_FREE_SPACE
        GROUP  BY TABLESPACE_NAME) F,
       (SELECT DD.TABLESPACE_NAME,
               Round(Sum(DD.BYTES) / ( 1024 * 1024 ), 2) TOT_GROOTTE_MB
        FROM   SYS.DBA_DATA_FILES DD
        GROUP  BY DD.TABLESPACE_NAME) D
WHERE  D.TABLESPACE_NAME = F.TABLESPACE_NAME
ORDER  BY 1

按仅限分：排它锁、共享锁      按资源分：数据锁、字典锁、内部锁、分布锁、并行缓冲管理锁
当语句修改一行时，事务仅获得改行的锁，通过在行级别锁定表数据，数据库最小化对同样数据的争用；
如果一个事务正在修改一行，那么行锁会阻止其他事务同时对改行的修改；
读操作从不阻塞写操作，除了select…for update语句；
写操作从不阻塞读操作，当行正被写操作修改时，数据库使用undo数据来给读操作提供行的一致性视图；
排它锁第一个单独锁定资源的事务是唯一可以改变该资源的事务，直到排它锁被释放；持有共享锁可以防止需要排它锁的写操作并发访问
当数据对象被加上排它锁时，其他的事务不能对它读取和修改；加了共享锁的数据对象可以被其他事务读取，但不能修改
Describe v$lock;
SID:会话标识符
TYPE:
  Tx事务
  TM DML或表锁
  MR介质恢复
  ST磁盘空间事务
LMODE/REQUEST:包含锁的模式
  0：无    1：空    2：行共享（RS）    3:行独占（RX）    4：共享（S）    5：共享行独占（SRX）    6:独占(X)
  若LMODE列含有一个不是0或1的整数，则表明进程已经获得一个锁。若REQUEST列含有一个不是0或1的数值，则表明进程正在等待一个锁，若LMODE列含胡数值0，则表明进程正等待获得一个锁

创建范围分区表
create table <表名>(<列名 数据类型,><列名 据类型,><列名 据类型,>)
  partition by range(<列名>)     //表示创建范围分区表
  (
    parttion p1 values less than(to_date('2022-02-01','YYYY-MM-DD')),  //小于2月份放在p1区
    parttion p2 values less than(to_date('2022-03-01','YYYY-MM-DD')),  //小于3月份放在p2区
	...
	parttion p_max values less then(maxvalue)     //不在上面分区的都存放在此分区
  )

创建列表分区表
create table <表名>(<列名 数据类型,><列名 数据类型,><列名 数据类型,>)
  partition by list(<列名>)     //表示创建列表分区表
  (
    parttion p1 values (591,590)),  //当列值为591或590时放在p1区
    parttion p1 values (592)),  //当列值为592时放在p2区
   	...
	parttion p1 values (DEFAULT)),  //当列值不为上面的值时放在此区
  )
  
创建散列分区表
create table <表名>(<列名 数据类型,><列名 数据类型,><列名 数据类型,>)
  partition by hash(<列名>)     //表示创建散列分区表
  PARTITIONS 12      //表示创建12个分区，建议分区的数量采用2的n次方，这样可以使得各个分区间数据分布更加均匀
  [store in(ts1,ts2,t3s...)]   //表示不同分区放在不同的表空间中
  
创建组合分区表
create table <表名>(<列名 数据类型,><列名 数据类型,><列名 数据类型,>)
  partition by range(<列名>)
    subpartition by TEMPLATE
	(
	  subpartition p_591 values (591),
	  subpartition p_592 values (592),

	 (
	   parttion p1 values less than(to_date('2022-02-01','YYYY-MM-DD')),
	   parttion p2 values less than(to_date('2022-03-01','YYYY-MM-DD')),
	   parttion p_max values less than(maxvalue)

	)

索引组织表最大的特点就是表就是索引，索引就是表，这种设计 一般适用在很少更新，频繁读的应用场合。
索引中不能存储空值，因此在允许非空的列上建索引要注意空值情况，以避免全局扫描
索引中实际数据值都存在于叶子块中，茎块和根块存储的都是管理信息

索引回表读一次只能读取一行，而全局扫描一次可以读取多行，因此在有大量回表读的情况下系统可能选择走全局扫描

索引执行计划：index range scan   //在B-tree不高且查询列较少的情况下进行范围扫描
			  index full scan(max/min)   //查询最大最小值时扫描
              index fast full scan   //此索引可一次读取多个索引块，不容易保证有序读取，但读取速度更快
			  index full scan   //两个full scan都是从头到尾遍历索引，此索引一次只读取一个块，在有排序动作时执行
			  table access by index rowid   //回表执行计划


如果在A字段查询返回的记录比较多，在B字段查询返回的记录也比较多，但同时查A、B字段返回的记录比较少，就适合建联合索引，联合索引不超过3个字段
如果单独条件查询返回的记录和联合条件查询返回的记录差别不大，则适合单独建索引，以减小索引段的大小
在等值查询情况下，组合索引的列无论哪一列在前，性能都一样
组合索引的两列，当一列是范围查询，一列是等值查询的情况下，等值查询列在前，范围查询列在后，这样的索引都是最高效
如果单列的查询列和联合索引的前置列一样，那单列可以不建索引，直接利用联合索引来进行检索数据

建索引的过程会产生锁，会把整个表锁住

在非空列有BTREE索引的情况下，一般用到该索引性能远高于全表扫描，不过性能最高的却是列上有位图索引的情况，甚至比用到普通非空列的BTREE索引时的性能又高出一大截，位图索引可以存储空值
位图索引满足两个条件：一是位图索引列大量重复，二是该表极少更新

查询条件中包含函数会用不到列的索引，此时需要建函数索引，其性能在普通索引与全表扫描之间，因尽量避免在条件语句中使用列函数运算

嵌套循环连接和哈希连接有驱动顺序，驱动表的顺序不同将影响表连接的性能，而排序合并转接没有驱动的概念，无论哪张表在前都无妨
嵌套循环连接要特别注意驱动表的顺序，小的结果集先访问，大的结果集后访问，才能保证被驱动表的访问次数降到最低，从而提升性能
除了嵌套循环、哈希连接连接外，排序合并需要排序
哈希连接不支持不等值连接<不支持>和<的连接方式，也不支持LIKE的连接方式
排序合并连接不支持不等值<>连接，也不支持LIKE的连接方式，但支持>之类的连接方式，嵌套循环就是支持所有的SQL连接条件写法，没有任何限制
嵌套循环连接最好驱动表的限制条件所在的列有索引，被驱动表的连接条件所在的列有索引。
在嵌套循环连接中，驱动表返回多少条记录，被驱动表就访问多少次
索引对于嵌套循环连接来说，既要考虑驱动表的限制条件，又要考虑驱动表上的连接条件上的索引，对于哈希连接来说仅仅考虑限制条件上的索引是否能用上索引，连接索引是不能发挥作用的，排序合并连接的连接条件虽然没有检索的作用，却有消除排序的作用

三．锁和数据库对象
1.锁：数据库用来控制共享资源并发访问的机制。
锁的类型：行级锁，表级锁
行级锁：对正在被修改的行进行锁定。行级锁也被称之为排他锁。
在使用下列语句时，Oracle会自动应用行级锁：
insert,update,delete,select…… for update
select……for update允许用户一次锁定多条记录进行更新。
使用commit or rollback释放锁。
表级锁：
lock table user_tbl in mode mode;
表级锁类型：
行共享 row share
行排他 row exclusive
共享 share
共享行排他 share row exclusive
排他 exclusive
死锁：两个或两个以上的事务相互等待对方释放资源，从而形成死锁
--锁类型：
  0：none     一般的SELECT，在表和行上都是0级锁
  1：null 空    SELET有时会在v$locked_object出现
  2：Row-S 行共享(RS)：共享表锁，sub share     不影响后一个2、3、4、5级锁，6级锁会报错
  3：Row-X 行独占(RX)：用于行的修改，sub exclusive      有insert,update,delete,select for update,lockrow exclusive,4、5、6级锁报错
  4：Share 共享锁(S)：阻止其他DML操作，share    create index,lockshare，阻止其它DML操作
  5：S/Row-X 共享行独占(SRX)：阻止其他事务操作，有主外键约束时updata/delete...待可能会产生4/5级锁
  6：exclusive 独占(X)：独立访问使用，exclusive   drop table,drop index,alter table,truncate table,lovk exclusive产生6级锁

五、游标管理
游标类型：隐式游标，显式游标，REF游标
REF游标用于处理运行时才能确定的动态SQL查询的结果

==========隐式游标==========
在PL/SQL中使用DML语句时自动创建隐式游标
隐式游标自动声明、打开和关闭，其名为SQL
隐式游标的属性：
%found SQL语句影响实质后返回true
%notfound SQL语句没有影响实质后返回true
%rowcount SQL语句影响的行数
%isopen 游标是否打开，始终为false
示例：
begin
update user_tbl set score=score+5;
if SQL%found then
dbms_output.put_line('数据被更改： '||SQL%rowcount);
elsif sql%notfound then
dbms_output.put_line('没有找到数据！');
end if;
if SQL%isopen then
dbms_output.put_line('Open');
else
dbms_output.put_line('Close');
end if;
end;

==========显式游标==========
在PL/SQL的声明部分定义查询，该查询可以返回多行
J 声明游标
J 打开游标
J 从游标中取回数据
J 关闭游标
声明游标完成两个任务：
给游标命名
将一个查询与游标关联
cursor cursor_name is select statement;
打开游标：
open cursor_name;
取数据：
fetch cursor_name into record_list;
关闭游标：
close cursor_name;
显式游标的属性：
%found 执行最后一条fetch语句成功返回行时为true
%notfound 执行最后一条fetch语句未能返回行时为true
%rowcount 返回到目前为止游标提取的行数
%isopen 游标是否打开

示例：
declare
users user_tbl%rowtype;
cursor boys_cur is select * from user_tbl where sex='h';
begin
open boys_cur;
loop
fetch boys_cur into users;
exit when boys_cur%notfound;
dbms_output.put_line(users.user_name||' '||users.password);
dbms_output.put_line(boys_cur%rowcount);
end loop;
close boys_cur;
end;

declare

六．子程序
子程序分为：存储过程和函数，它是命名的PL/SQL块，编译并存储在数据库中。
子程序的各个部分：声明部分，可执行部分，异常处理部分。
过程----执行某些操作
函数----执行操作并返回值

==========存储过程==========
创建过程的语法：
create or replace procedure
proce_name (parameter_list)
is|as
local variable declaration
begin
executable statements
exception
exception_handlers
end proce_name;

过程参数的三种模式：
In----用于接收调用的值，默认的参数模式
Out----用于向调用程序返回值
In out----用于接收调用程序的值，并向调用程序返回更新的值
执行过程的语法：
Execute proce_name(parameter_list);
或
Declare
Variable var_list;
Begin
Proce_name(var_list);
End;
将过程执行的权限授予其他用户：
Grant execute on proce_name to scott;
Grant execute on proce_name to public;
删除存储过程：
Drop procedure proce_name;

==========函数==========
创建函数的语法：
Create or replace function
Fun_name (parameter_list)
Return datatype is|as
Local declarations
Begin
Executable statements;
Return result;
Exception
Exce_handlers;
End;
函数只能接收in参数，不能接受out或in out参数，形参不能是PL/SQL类型
函数的返回类型也必须是数据库类型
访问函数的方式：
J 使用PL/SQL块
J 使用SQL语句
Select fun_name(parameter_list) from dual;
