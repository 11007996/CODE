## 概述

LuxVideoDet 基于 **.NET 8**，核心库为 **LuxVideoDet.Core**；宿主程序包括 **Desktop**（Avalonia）、**Web**、**Headless**。同一套代码在不同操作系统上需满足对应原生依赖（OpenCV、可选 CUDA/OpenVINO 等），详见下文矩阵与各节。

---

## 平台与能力矩阵

| 组件 | Windows | Linux | macOS | 说明 |
|------|---------|-------|-------|------|
| .NET 8.0 | ✅ | ✅ | ✅ | 需安装 [SDK](https://dotnet.microsoft.com/download/dotnet/8.0)（开发构建）或 Runtime（仅运行发布包） |
| OpenCvSharp / 图像 | ✅ | ✅（通常需系统 OpenCV） | ✅ | Linux 常见依赖 `libopencv-dev`；macOS 可用 Homebrew `opencv` |
| ONNX Runtime（CPU） | ✅ | ✅ | ✅ | 默认构建 |
| ONNX Runtime GPU（CUDA） | ✅ | ✅ | ❌ | 构建加 `-p:UseCUDA=true`；需 NVIDIA 驱动与 CUDA 环境 |
| ONNX Runtime QNN | ❌ | ✅（ARM64 等） | ❌ | 构建加 `-p:UseQNN=true`；需 QNN SDK 与文档中的附加属性 |
| OpenVINO | ✅ | ✅ | ⚠️ | macOS 需自行安装；运行时选择 `InferenceDevice.OpenVINO` |
| SkiaSharp（中文等绘制） | ✅ | ✅ | ✅ | Linux 建议安装中文字体包 |

## 通用前置条件

- **.NET 8**：开发机安装 SDK；生产环境可只装 [Runtime](https://dotnet.microsoft.com/download/dotnet/8.0)（若使用自包含发布则按发布方式决定）。
- **克隆与还原**：

```bash
git clone <repository-url>
cd LuxVideoDet
dotnet restore
```

## 构建与推理后端选项

以下命令均在仓库根目录执行。

| 场景 | 命令 |
|------|------|
| 默认（ONNX CPU） | `dotnet build -c Release` |
| CUDA（Windows / Linux x64，需对应环境） | `dotnet build -c Release -p:UseCUDA=true` |
| QNN（Linux ARM64 等，需 QNN 与 ORT 路径） | `dotnet build -c Release -p:UseQNN=true` 并配合 `LuxVideoDet.Core.csproj` 注释中的 `QnnOrtSourceDir`、`QnnSdkLibDir` 等属性 |

**说明**：运行应用时若使用 GPU 构建产物，请对 **`dotnet run` / `dotnet publish` 使用相同 `-p:UseCUDA=true`**，否则可能仍链接 CPU 版 ONNX Runtime。

## 按操作系统部署

### Windows

**环境**

- Windows 10/11（x64）
- .NET 8 SDK 或 Runtime

**示例（PowerShell）**

```powershell
winget install Microsoft.DotNet.SDK.8
# 或仅 Runtime：winget install Microsoft.DotNet.Runtime.8

cd LuxVideoDet
dotnet build -c Release
dotnet run --project LuxVideoDet.Headless
```

**可选：NVIDIA CUDA**

1. 安装与驱动匹配的 [CUDA Toolkit](https://developer.nvidia.com/cuda-downloads)。
2. `dotnet build -c Release -p:UseCUDA=true` 后再运行目标项目。

---

### Linux

**环境**

- 常见发行版：Ubuntu 22.04 / Debian 12 / RHEL 9 等（以实际测试为准）
- .NET 8、OpenCV 开发包及基础构建依赖

**示例（Ubuntu/Debian）**

```bash
# .NET（官方脚本示例；也可用包管理器安装 dotnet-sdk-8.0）
wget https://dot.net/v1/dotnet-install.sh
chmod +x dotnet-install.sh
./dotnet-install.sh --channel 8.0
# 将 dotnet 加入 PATH 后：

sudo apt update
sudo apt install -y libopencv-dev libgdiplus libc6-dev

cd LuxVideoDet
dotnet build -c Release
dotnet run --project LuxVideoDet.Headless
```

**可选：OpenVINO**

若使用 OpenVINO 推理，请按 [Intel OpenVINO](https://docs.openvino.ai/) 安装 Runtime，并配置 `setupvars.sh` 等环境变量；配置中设备选择 `OpenVINO`（见 [配置说明](#配置说明)）。

**可选：NVIDIA CUDA**

```bash
# 示例：按发行版安装 NVIDIA 驱动与 CUDA，然后
dotnet build -c Release -p:UseCUDA=true
```

**可选：Qualcomm QNN（ARM64）**

仅适用于目标设备与 SDK 支持的场景，构建参数与路径以 **`LuxVideoDet.Core.csproj`** 内注释为准。

---

### macOS

**环境**

- macOS 12（Monterey）或更高（建议与项目实际测试版本一致）
- .NET 8、Homebrew（推荐）

**示例**

```bash
brew install dotnet@8
brew install opencv

cd LuxVideoDet
dotnet build -c Release
dotnet run --project LuxVideoDet.Headless
```

**OpenVINO**：官方对 macOS 支持有限，一般优先使用 **ONNX Runtime CPU**；若需 OpenVINO，可尝试 `brew install openvino` 并自行验证与 `OpenVINO.CSharp.API` 的兼容性。

## Docker（容器）

以下为最小示例：基于官方 .NET 镜像、安装 OpenCV 相关依赖后发布 **Headless**（请按实际端口与挂载路径调整）。

```dockerfile
FROM mcr.microsoft.com/dotnet/aspnet:8.0 AS base
WORKDIR /app
RUN apt-get update && apt-get install -y \
    libopencv-dev libgdiplus libc6-dev \
    && rm -rf /var/lib/apt/lists/*

FROM mcr.microsoft.com/dotnet/sdk:8.0 AS build
WORKDIR /src
COPY . .
RUN dotnet restore
RUN dotnet publish LuxVideoDet.Headless/LuxVideoDet.Headless.csproj -c Release -o /app/publish

FROM base AS final
WORKDIR /app
COPY --from=build /app/publish .
ENTRYPOINT ["dotnet", "LuxVideoDet.Headless.dll"]
```

构建与运行示例：

```bash
docker build -t luxvideodet:latest .
docker run -d --name luxvideodet \
  -v /path/to/config:/app \
  -v /path/to/models:/models \
  luxvideodet:latest
```

挂载路径、环境变量与工作目录需与 **Headless** 读取配置的方式一致（如 `configs.json` 工作目录）。

## 配置说明

各平台共用 **JSON** 配置（默认由 Core 的 `SingleFileConfigurationStore` 使用 **`configs.json`** 等，详见 `LuxVideoDet.Core/README.md`）。推理相关字段在单算法配置 **`AlgorithmConfig.Inference`** 中，例如：

```json
{
  "inference": {
    "modelPath": "models/yolov8n.onnx",
    "device": "CPU",
    "confidenceThreshold": 0.5,
    "iouThreshold": 0.45,
    "modelType": "Auto",
    "classes": ["person", "car"]
  }
}
```

**`device`（`InferenceDevice`）与实现关系简述**：

| 值 | 说明 |
|----|------|
| `CPU` | ONNX Runtime CPU，全平台 |
| `GPU` | ONNX Runtime GPU（CUDA），需 CUDA 构建与驱动 |
| `OpenVINO` | OpenVINO 引擎加载 `.onnx` / `.xml` 等 |
| `QNN` | 限定平台与构建 |
| `CoreML` | Apple 平台 ONNX Runtime CoreML EP（按项目支持情况） |

引擎具体选型由 **`InferenceEngineFactory`** 与配置共同决定，以源码为准。

## 故障排查

### 找不到 OpenCV（Linux）

```bash
sudo apt install libopencv-dev
export LD_LIBRARY_PATH=/usr/lib/x86_64-linux-gnu:$LD_LIBRARY_PATH
```

### 找不到 OpenCV（macOS）

```bash
brew install opencv
export DYLD_LIBRARY_PATH=/opt/homebrew/lib:$DYLD_LIBRARY_PATH
```

### SkiaSharp / 中文显示异常（Linux）

```bash
sudo apt install fonts-noto-cjk
fc-cache -fv
```

### OpenVINO 初始化失败

检查是否已 `source` 安装目录下的 `setupvars.sh`（路径因安装方式而异），以及 `OPENVINO_DIR` 等变量是否与当前 shell 一致。

## 性能与资源

- **NVIDIA**：使用 CUDA 构建时保证驱动与运行时匹配；笔记本注意电源与散热策略。
- **Intel**：可在支持的硬件上评估 **OpenVINO**。
- **ARM + Qualcomm**：在支持设备上评估 **QNN** 构建。
- **macOS**：以 ONNX Runtime CPU/CoreML 为主；线程数等见 `InferenceConfig.ThreadCount` 与产线文档。
