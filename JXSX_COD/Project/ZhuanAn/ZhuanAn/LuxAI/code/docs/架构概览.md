# LuxVideoDet 架构概览

本文说明解决方案的分层与模块职责。**实现细节以当前源码为准**；类名、工厂注册方式等若有变更，请直接查阅 `LuxVideoDet.Core` 与各宿主项目。

更细的交付与扩展步骤见 [产线交付与二次开发指南](PRODUCTION_HANDOFF.md)；Core 内目录说明见 [LuxVideoDet.Core/README.md](../LuxVideoDet.Core/README.md)。

---

## 解决方案组成

| 项目 | 角色 |
|------|------|
| **LuxVideoDet.Core** | 跨平台核心库：配置、推理、视频源、算法与流水线、存储、通知、区域、AOI、日志抽象等 |
| **LuxVideoDet.Desktop** | 桌面宿主：**Avalonia** UI，引用 Core，负责配置界面与实时预览 |
| **LuxVideoDet.Web** | Web 宿主：API / Blazor / SignalR 等（按项目内实际代码） |
| **LuxVideoDet.Headless** | 无界面宿主：适合产线或后台常驻 |
| **LuxVideoDet.Tests** | 单元 / 集成测试 |

宿主程序通过 `AddLuxVideoDetCore()`（见 `LuxVideoDet.Core/Extensions/ServiceCollectionExtensions.cs`）注册配置服务、各类工厂（推理、视频源、算法、后处理、通知、AOI、流水线等）。

---

## 分层关系

```
┌─────────────────────────────────────────────┐
│  宿主（Desktop / Web / Headless）           │
│  启动、配置 UI 或 API、进程生命周期           │
├─────────────────────────────────────────────┤
│  LuxVideoDet.Core                           │
│  检测流水线、推理、视频、存储、通知、配置     │
└─────────────────────────────────────────────┘
```

- **Core** 不依赖具体 UI 框架；桌面/Web/无头仅负责装配与展示。
- 可配置组件（推理引擎、算法类型、视频源、通知通道）多通过 **工厂** 按 `DetectionConfiguration` / 子配置创建。

---

## Core 内部模块（逻辑视图）

| 模块 | 职责摘要 |
|------|----------|
| **Configuration** | 强类型 `DetectionConfiguration`、JSON 存储（默认单文件 `configs.json`）、`ConfigurationService`、FluentValidation |
| **Inference** | `IInferenceEngine`；ONNX Runtime / OpenVINO 实现；`InferenceEngineFactory`；按模型任务选择 **Postprocessor**（检测、分割、OBB、姿态、分类等） |
| **VideoSource** | `IVideoSource`；本地文件、RTSP、摄像头等；`VideoSourceFactory` |
| **Algorithm** | `IDetectionAlgorithm`、**同步** `Process(Frame)`；`DetectionAlgorithmBase`（模板方法）；各业务在 `Implementations/*`；`DetectionAlgorithmFactory` 通过 **`IAlgorithmDescriptor` 反射注册**，非手写大段 switch |
| **Pipeline** | `PipelineFactory` 编排「视频源 + 算法」；`AlgorithmWorker`、`MultiAlgorithmPipeline` 等（以源码为准） |
| **Region / Aoi** | 区域与 AOI 检测器（如 U 型），`AoiDetectorFactory` |
| **Storage** | 错图、再训练图、录像、统计等 |
| **Notification** | Webhook、微信、PLC、扬声器等；`NotificationServiceFactory` |
| **Logging** | `Microsoft.Extensions.Logging` + Serilog；`ILogEventSink` 供桌面订阅 |
| **Rendering** | 检测结果叠加绘制等（与 UI 解耦的绘制辅助） |
| **Extensions** | `AddLuxVideoDetCore`、`AddSerilogLogging`、`AddDesktopLogging` |

完整文件夹树见 [LuxVideoDet.Core/README.md](../LuxVideoDet.Core/README.md#代码目录)。

## 典型数据流

```
视频源 ReadFrame → 算法 Process(Frame)
                      ├─ 推理 IInferenceEngine + Postprocessor
                      ├─ 区域 / AOI / 业务状态机（各算法可选）
                      ├─ 存储 / 录像
                      └─ 通知
```

桌面端在算法返回后做 **RenderAnnotations** 与界面刷新；Headless 可无渲染仅日志与通知。具体线程模型与是否多算法并行以 **Pipeline 实现** 为准。

## 三种宿主差异（概念）

| 维度 | Desktop | Web | Headless |
|------|---------|-----|----------|
| 界面 | Avalonia 实时配置与预览 | 浏览器 / API | 无 |
| Core 用法 | 同左，附加桌面日志、UI 命令 | 同左，侧重服务与远程 | 同左，侧重常驻与资源占用 |

运行与部署细节见 [应用模式说明](APPLICATION_MODES.md)。
