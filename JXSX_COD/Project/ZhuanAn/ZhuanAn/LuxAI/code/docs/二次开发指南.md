## 共性约定（三种扩展都适用）

1. **反射注册，不改工厂里的手写分支**  
   `DetectionAlgorithmFactory`、`NotificationServiceFactory`、`AoiDetectorFactory` 均在启动时扫描程序集中对应 **Descriptor** 接口的实现并注册。

2. **示例模板不参与注册**  
   带有 **`[ExampleTemplate]`**（`ExampleTemplateAttribute`）的 Descriptor **不会**被注册。复制示例到自有目录后**去掉该特性**，才会生效。

3. **类型键统一小写**  
   配置 JSON、工厂查找均按 **小写** 匹配（如 `algorithmType`、`notifiers[].type`、AOI 的 `detectorName`）。

## 一、算法（Detection Algorithm）增量开发

### 1.1 要做什么

新增一种可在配置里通过 `algorithmType` 选择的业务算法：通常包括 **状态/规则逻辑**（继承 `DetectionAlgorithmBase`）+ **元数据**（实现 `IAlgorithmDescriptor`）。

### 1.2 文件与目录

在 **`LuxVideoDet.Core/Algorithm/Implementations/<你的算法目录>/`** 下放置：

| 文件 | 职责 |
|------|------|
| `*Algorithm.cs` | 继承 **`DetectionAlgorithmBase`**，实现 **`ProcessDetections`**、**`Reset`**、**`GetCurrentState`** 等；推理、存图、通知、性能日志等多由基类统一处理。 |
| `*Descriptor.cs` | 实现 **`IAlgorithmDescriptor`**：`TypeKey`、`DisplayName`、`DefaultClasses`、`RequiredRegions`、`DefaultClassColors`，以及在 **`Create`** 里 `new` 你的算法类。 |

可复制 **`Implementations/Example/`** 作为起点，上线前去掉 **`[ExampleTemplate]`**。

### 1.3 `IAlgorithmDescriptor` 要点

- **`TypeKey`**：与配置 **`algorithmType`** 一致（小写），例如 `midframescan`、`u7lite`。
- **`Create(..., IServiceProvider serviceProvider)`**：除 `IInferenceEngine`、`RegionManager`、通知列表外，若需要 **AOI**、其它单例服务，可在此 **`serviceProvider.GetRequiredService<T>()`** 并传入算法构造函数（参见 **U7Lite** 与 `AoiDetectorFactory`）。
- **`SupportedNotifierTypes`**（可选）：限制本算法在 UI 中可选的通知 `type`；`null` 或空表示不限制。
- **`GetDefaultNotifierParameters(notifierType)`**（可选）：为某通知类型提供默认参数，与 JSON 合并时**配置覆盖默认**。
- **`GetUiActionDefinitions()`**（可选）：声明实时画面上的按钮；算法宜实现 **`IAlgorithmUiCommandHandler`** 处理点击。

### 1.4 配置侧

在 **`DetectionConfiguration.Algorithms[]`** 中增加一项，`algorithmType` 等于你的 **`TypeKey`**，并补全 `inference`、`regions`、`notification` 等。无需修改 **`DetectionAlgorithmFactory.cs`**。

### 1.5 可选能力速查

| 需求 | 说明 |
|------|------|
| 画面按钮 | `IAlgorithmUiCommandHandler` + Descriptor 中 `GetUiActionDefinitions`，见 **DESCRIPTOR.md** |
| 产量/OK·NG 统计 | 可选实现 **`IProductionStatisticsProvider`**（供 Desktop 等展示计数） |

## 二、通知方式（Notification Channel）增量开发

### 2.1 要做什么

新增一种可在 **`notification.notifiers[].type`** 中填写的渠道（如 `webhook`、`speaker`、`plc`）：需要 **`INotificationService` 实现** + **`INotificationDescriptor` 实现**。

### 2.2 文件与目录

在 **`LuxVideoDet.Core/Notification/<渠道目录>/`** 下建议成对存在：

| 文件 | 职责 |
|------|------|
| `*NotificationService.cs` | 实现 **`INotificationService`**：`NotificationType`、`Initialize(Dictionary<string, object>)`、`SendAsync(...)`。初始化失败时（缺必填参数）应抛 **`ArgumentException`**，工厂会捕获并**跳过该渠道**（日志中有「未加入列表」说明）。 |
| `*NotificationDescriptor.cs` | 实现 **`INotificationDescriptor`**：`TypeKey`（小写）、`DisplayName`、**`ParameterDefinitions`**（供 Desktop/Web 表单与校验）、**`Create(NotificationServiceFactoryContext)`** 返回未 Initialize 的服务实例。 |

参考现有：**`Notification/Webhook/WebhookNotificationDescriptor.cs`** 与对应 `WebhookNotificationService`。

### 2.3 `NotificationServiceFactoryContext`

Descriptor 的 **`Create`** 会收到上下文（含 **`ILoggerFactory`**、**`IHttpClientFactory`** 等），用于构造服务；**不要**在 Descriptor 里调用 `Initialize`，由工厂统一 `Initialize(config)`。

### 2.4 与算法配置的关系

- 全局已注册的通知类型由 **`NotificationServiceFactory.GetSupportedTypes()`** 列出。
- 某算法若只想开放部分渠道，在该算法的 **`IAlgorithmDescriptor.SupportedNotifierTypes`** 中列出允许的 **`type`** 键。
- 算法 Descriptor 的 **`GetDefaultNotifierParameters`** 可与 JSON 参数合并（**配置优先**），见 **`DetectionAlgorithmFactory`** 内 `MergeNotifierParameters`。

### 2.5 配置示例（概念）

```json
"notification": {
  "enabled": true,
  "notifiers": [
    { "type": "webhook", "enabled": true, "parameters": { "businessWebhook": "https://..." } }
  ]
}
```

**无需修改 `NotificationServiceFactory.cs`**（反射注册新 Descriptor）。

## 三、AOI（传统视觉 / 几何检测器）增量开发

### 3.1 要做什么

AOI 与「深度学习推理」分离：在 **`Mat` ROI** 上做几何/传统算法，供业务算法在 **`ProcessDetections`** 前后调用。扩展方式为 **`IAoiDetector`** + **`IAoiDetectorDescriptor`**。

### 3.2 接口

- **`IAoiDetector`**：`Name`、`Initialize`、**`Detect(Mat roi, ...)`** / **`DetectBatch`**。
- **`IAoiDetectorDescriptor`**：`TypeKey`、**`Aliases`**（别名，与 `TypeKey` 等价注册）、`DisplayName`、**`Create(ILogger)`**。

### 3.3 文件与目录

在 **`LuxVideoDet.Core/Aoi/<你的模块>/`** 下放置检测器实现与 **`XxxDescriptor.cs`**。参考 **`Aoi/UDet/`**（`UShapeDetector` + `UShapeDetectorDescriptor`，`TypeKey` 为 `u_shape`，别名 `udet`）。

### 3.4 注册方式

1. **反射（常规）**：无 **`[ExampleTemplate]`** 的 **`IAoiDetectorDescriptor`** 在进程启动时自动注册；**`TypeKey` 与 `Aliases` 都会映射到同一 Descriptor**。
2. **运行时补充**：**`AoiDetectorFactory.Register(name, factory)`** 用于测试或动态注册（一般少用）。

### 3.5 在检测算法中使用

算法**不**直接 new AOI 类，而是通过 **`AoiDetectorFactory`**：

1. 在 **`IAlgorithmDescriptor.Create`** 中从 **`serviceProvider.GetRequiredService<AoiDetectorFactory>()`** 取出工厂。
2. 将工厂传入算法构造函数（参见 **`U7LiteDescriptor`** / **`U7LiteAlgorithm`**）。
3. 在 **`Initialize` 或首次使用前** 调用 **`CreateDetector("u_shape")`**（或你的 `TypeKey`），再 **`Initialize(...)`** 与业务参数。

**无需修改 `AoiDetectorFactory.cs`**（除非你要改全局注册策略）。
